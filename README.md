# Проект: Модуль работы с комплексными числами (`ComplexNumber`)

## Назначение
Программный модуль на Python для создания, обработки и тестирования комплексных чисел вида *a + bi*. Включает реализацию арифметических операций, вычисления модуля, вспомогательных функций и полного набора тестов.

## Функциональные возможности
- **Создание** комплексных чисел с действительной и мнимой частями.
- **Арифметические операции**:
  - сложение (`+`);
  - вычитание (`−`);
  - умножение (`×`).
- **Вычисление модуля** по формуле $\sqrt{a^2 + b^2}$.
- **Поиск числа с максимальным модулем** в списке (с обработкой пустого списка).
- **Форматированный вывод** числа и его компонентов.
- **Обработка ошибок** типов и граничных случаев.

## Архитектура решения

### Основной класс
**`ComplexNumber`** — реализует представление комплексного числа.
**Ключевые методы**:
- `__init__(real, imaginary)` — инициализация;
- `__add__(other)` — сложение с проверкой типа;
- `__sub__(other)` — вычитание с проверкой типа;
- `__mul__(other)` — умножение по математической формуле;
- `__str__()` — строковое представление с корректным знаком мнимой части;
- `magnitude()` — вычисление модуля.

### Вспомогательные функции
- `find_largest_magnitude(numbers_list)` — поиск числа с наибольшим модулем;
- `print_summary(real_part, imag_part)` — вывод компонентов числа.

## Тестирование
Реализован полный набор **юнит‑тестов** на базе `unittest`:
- **позитивные тесты** — проверка корректной работы при валидных данных;
- **негативные тесты** — обработка некорректных входных данных;
- **граничные случаи** — работа с нулевыми значениями и пустым списком;
- **тестирование «белым ящиком»** — покрытие всех ветвлений кода;
- **тестирование «чёрным ящиком»** — проверка функциональности без учёта реализации.
